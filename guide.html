GETTING STARTED

LayCAT はじめ方ガイド

初めて LayCAT を触る方向けの入門ガイドです。アクセスからプロジェクト作成、動画レビュー、提出、通知、メンバー招待までを、上から順に読むだけで一通り使えるようになります。

00LayCAT とは？

LayCAT（レイキャット）は、映像制作のレビューと進行管理をブラウザ 1 つで完結させるツールです。

  • 動画・画像をアップして、フレームごとに コメント・描き込みでフィードバック
  • ショット × 工程（LAY / ANIM / COMP など）で 進捗管理
  • 提出物を 「サブミット」でまとめて 送って一括レビュー
  • 単一 HTML（インストール不要）で動作。ローカルフォルダ運用もクラウド運用も両対応

「レビュー ＆ 進行管理ツール」というカテゴリの製品です。同種のツールとしては ShotGrid（Autodesk）・Frame.io（Adobe）・ftrack などがありますが、LayCAT は単一 HTML＋自前クラウドという軽量アーキテクチャで、個人〜小規模チームでも使いやすいのが特徴です。

01動作環境の確認

  • ブラウザ：Chrome または Edge を推奨（フォルダ直接編集に必要な File System Access API 対応が理由）。Safari / Firefox は R2 クラウド運用のみ対応
  • OS：Windows / macOS / Linux 全対応
  • 回線：R2 クラウド運用時は常時オンライン、フォルダ運用はオフラインでも動作
  • Google アカウント：R2 クラウド運用時のみ必要（ログインで本人確認）

ローカル HTML（file://）では動きません。必ずhttps:// の LayCAT サイトからアクセスしてください（File System Access API が https でしか有効化されないため）。

02アプリにアクセス

ブラウザで下のリンクを開くだけで起動します。インストール・ダウンロード不要です。

https://ogshaw03.github.io/lay_cat/

ブックマークに追加しておくと便利です。トップページ（intro.html）の右上「アプリを開く」ボタンからも起動できます。

初回起動時は少し重いですが、2 回目以降はブラウザキャッシュから即座に開きます。

03ログイン（クラウド運用時のみ）

LayCAT には認証が有効な運用と無効な運用の 2 パターンがあります。

  • 認証あり（R2 クラウド運用・複数人共有）：起動時にログイン画面が出ます。「Google でログイン」を押して、事前に運営が承認したメールアドレスでサインインしてください
  • 認証なし（ローカル運用・自分だけで使う）：ログイン画面は出ず、そのままアプリが起動します

「アクセス権がありません」と出た場合：運営（プロジェクト所有者）に、あなたの Google アカウントのメールアドレスをアクセス許可リストに追加してもらってください。

04プロジェクトを作成

ホーム画面右上の 「＋ プロジェクトを作成」 から作成できます。保存先を 2 択で選ぶのがポイントです。

📁 フォルダ運用 RECOMMENDED for 個人・小規模

  • PC のローカルフォルダ、または Google Drive / Dropbox の同期フォルダにデータを保存
  • ログイン不要、追加費用ゼロ
  • Chrome / Edge で File System Access API を使ってフォルダを直接読み書き
  • 他の人と共有するときは、そのフォルダを共有ドライブ経由で共有

☁️ クラウド運用（R2）RECOMMENDED for チーム

  • データは Cloudflare R2（オブジェクトストレージ）に保存
  • Google ログイン必須（誰がアクセスできるかを R2 側で管理）
  • ブラウザの種類・OS を問わず同じデータを見られる
  • ネットさえあればどこからでも接続可

フォルダ運用でのプロジェクト作成

  1. 「＋ プロジェクトを作成」 → 「フォルダに保存」 を選択
  2. 「フォルダを選ぶ」ボタンでプロジェクト用の空フォルダを指定（例：D:\projects\新作アニメ_ep01）
  3. プロジェクト名を入力して「作成」
  4. そのフォルダの中に laycat.project.json が作られ、これが LayCAT の実体になります

R2 クラウド運用でのプロジェクト作成

  1. 「＋ プロジェクトを作成」 → 「クラウドに保存（R2）」 を選択
  2. プロジェクト名を入力して「作成」
  3. あなたが所有者（owner）として自動登録されます。他メンバーは後述の手順で招待できます

迷ったら「フォルダ運用」からスタートするのがおすすめです。あとで気に入ったら R2 に移行できます。まずは触ってみて、複数人共有が必要になった時点で R2 を検討してください。

05カット・工程を作る

LayCAT のデータ構造は 3 階層です：

プロジェクト
  └─ カテゴリ（EP01 / SEQ_A など、任意で細分化）
       └─ ショット（SC001, SC002, ...）
            └─ 工程（LAY / ANIM / COMP など。＝「タスク」）

推奨の作り方

  1. プロジェクトを開いた画面で、まず カテゴリ（EP01 など） を追加
  2. そのカテゴリの「工程設定」で、このカテゴリ配下で使う工程テンプレート（例：LAY, ANIM, COMP）を先に登録しておく
  3. カテゴリの中に ショット（SC001…）を追加すると、工程テンプレートに沿って自動で全工程が作られる

ショット名は SC001, SC002 のようにゼロ埋めにしておくと、サイドバーで自然な順番に並びます（SC1 と SC10 だと SC10 が先に来てしまいます）。

06動画をアップロードする

  1. 左サイドバーで、動画を上げたい 工程（タスク） を選ぶ
  2. タスクページ右上の 「＋ 動画」 ボタンを押す
  3. ファイル選択画面で動画（mp4 / mov / webm 等）を選ぶ
  4. 動画名（省略可）とコメント（省略可）を入力して「アップロード」

Animation / ProRes コーデックはブラウザで再生できません。Maya / After Effects からの書き出しは H.264 の mp4 / mov にしてください。それ以外だと動画枠にコーデック警告オーバーレイが出ます。

複数版（バージョン）の扱い

同じタスクに再度アップロードすると、新しい版 として追加されます。過去版は履歴として残り、いつでも比較・遡って再生できます。ステータスは自動で「チェック待ち」に戻ります。

07レビュー画面の使い方

タスクページで動画タイルをクリック（または右上の ⤢ ボタン）で、フル画面のレビュー画面 が開きます。

再生と操作

Space再生 / 停止
← →1 フレーム前後
,IN 点をセット
.OUT 点をセット
X再生範囲を解除
V左右反転（鏡像）
Space+ドラッグパン（画面移動）
Ctrl+Space+右ドラッグズーム
Shift+Space+ドラッグ回転
Ctrl+Z / Y元に戻す / やり直し

ペン描画・消しゴム・トレペ・顔向きガイド

  • ◌ ペン：クリックで OFF → 描画 → 消しゴム と循環。サイズ・色・筆圧対応
  • トレペ：白レイヤーを重ねてトレース風に。25% ずつ濃度が変わる
  • ◑ 顔の向きガイド：正中線と目印マーカーで、3D 的な顔の向きを指示できる（アニメーターへの指示に便利）
  • ゴースト：前後フレームのアノテを半透明で重ねて、動きの補正指示がしやすい

フレームコメント

  • 再生を止めた位置で右下の入力欄に書き込むと、そのフレームに紐づいたコメントとして保存される
  • コメントをクリックすればそのフレームに一発ジャンプ
  • コメント内で @名前 でメンバーを呼ぶと、その人の通知に飛ぶ

比較モード（⇄ 比較）

版カードや別のタスクから「⇄ 比較」で選ぶと、レビュー画面が左右 2 分割になり、フレームを同期しながら見比べられます。修正前後の確認や、レイアウトと原画の突き合わせに便利。

音量・ミュート

再生バーの 🔊 / 🔇 ボタンと音量スライダで音声を調整できます。状態はブラウザに保存され、次回起動時も引き継がれます。

08ステータスと提出

各工程には ステータス があります（既定：チェック待ち → OK / リテイク / NG / オミット）。

ステータスの変え方

  • タスクページ上部の ステータス切替プルダウン から選択
  • レビュー画面の判定ボタン（〇 / △ / × / — 相当）からも変更可
  • プロジェクト設定でステータスの種類はカスタマイズ可能

「提出（チェック依頼）」の 2 通りの方法

  1. タスクページから直接：新しい版をアップ → 自動で「チェック待ち」に → チェック担当者に通知
  2. サブミット機能で複数まとめて：次のセクション参照

「チェック担当者」の設定は必須です。タスクページ上部の「チェック」プルダウンからメンバーを選んでおくと、その人に通知が飛びます。未設定だと誰にも通知が行きません。

09サブミット（複数まとめて提出）

「今週のレイアウト 8 カット提出」のように、複数ショット・工程をまとめて 1 つの束 として提出する仕組みです。

作り方

  1. ヘッダの 「サブミット」 ボタンを押す
  2. 「＋ ショットを追加」で、提出したい工程を選ぶ（作業担当者で絞り込み可）
  3. 各ブロックに動画をドロップ／選択、コメント・ステータスを設定
  4. 「サブミットを作成」で確定

受け取り側

プロジェクトタブの「サブミット」タブに新しいサブミットが一覧表示されます。ここから 1 カットずつレビュー画面に飛べます。チェック担当者には自動でチェック依頼通知が届きます。

サブミットは 「レビュー会前にまとめて渡したい」 ユースケース向けです。単発の版アップだけで済むならタスクページ直接アップで十分。

10通知

ヘッダ右側の 🔔 ベルアイコン をクリックで通知パネルが開きます。

通知が来るタイミング

  • @ メンション：コメントやフレームコメントで自分が @ で呼ばれた時
  • チェック依頼：自分が「チェック担当」に設定されているタスクに、他の人が新規版をアップした時（サブミット経由でもタスク直接アップでもどちらも通知）

いつ検知されるか

LayCAT は「リアルタイム push」ではなく、以下のタイミングでフォルダ or R2 の最新を取り込んで通知を再生成する仕組みです：

  • アプリを起動した時（＝リロード時）
  • 🔔 ベルアイコンを開いた時（フォルダから最新を再取得してからスキャン）
  • ベルパネルの「🗘 最新」ボタンを押した時

相手が動画を上げた瞬間には気づけません。作業を始める前にベルを開く／リロードするのを習慣にしておくと安心です。

11メンバー招待・アクセス制御

メンバーを追加

  1. プロジェクト設定 → 「メンバー管理」 を開く
  2. 「＋ メンバー追加」で、そのメンバーの Google アカウントのメールアドレス を入力
  3. 名前・ロール（director / animator など・任意）・色を設定して「保存」

ID には 必ずメールアドレス（Google アカウントと一致）を入れてください。これがログイン認証との紐付けの鍵になります。

R2 クラウド運用の場合の追加ステップ

R2 プロジェクトでは、メンバー登録に加えて Cloudflare Worker 側の _access.json にもメールが登録される必要があります（メンバー管理から保存すれば自動で書き込まれます）。招待されるメンバーは、事前に運営から Google ログインを許可してもらう必要があります。

担当割り当て

タスクページ上部の 「作業」「チェック」プルダウン でメンバーを割り当てると、そのタスクの担当が確定します。この設定に基づいて、通知の飛び先や「担当者別のチェック待ち一覧」が組まれます。

12EXR プレビュー（応用）

LayCAT は OpenEXR ファイル（.exr）をブラウザ内でプレビュー できる数少ないレビューツールです。CG コンポジターやビジュアルデベロップの現場で威力を発揮します。

できること

  • 単一 / 多層 EXR のパースと表示（RGBA, Depth, ID, AOV）
  • 露出（Exposure）・ガンマ のリアルタイム調整
  • Cryptomatte でオブジェクトを選択して抜き出し
  • Depth レイヤーを AE の EXtractoR 準拠で表示
  • アノテウィンドウのトップバーに全機能が集約（別モーダル無し）

PIZ 圧縮 EXR は現状未対応です。ZIP / ZIPS / PXR24 圧縮は対応済み。

13Q&A・トラブルシューティング

Q. 動画をアップしたのに再生できない

コーデックが Animation / ProRes などブラウザ非対応の可能性が高いです。H.264 の mp4 / mov に変換してから再アップしてください。

Q. 音が出ない

レビュー画面の再生バーに 🔊 / 🔇 トグルと音量スライダがあります。ミュートになっていないか、音量が 0 になっていないか確認してください。それでも出ない場合は元動画に音声トラックが含まれていない可能性があります。

Q. チェック依頼の通知が来ない

  1. タスクの 「チェック」担当者に自分が設定されているか確認
  2. アプリをリロードするか、🔔 ベル → 「🗘 最新」を押してみる
  3. それでも来なければ運営に相談（自分のメールがメンバー一覧に登録されているか）

Q. フォルダを開き直したらデータが古い

他の人が更新した内容は、手動でフォルダを再読み込みする必要があります。ヘッダの「🗘 最新」ボタン、または一度アプリをリロードしてください。

Q. プロジェクトを他人と共有したい（フォルダ運用）

Google Drive / Dropbox など同期フォルダにプロジェクトフォルダを置き、共有相手には同じフォルダをブラウザで「フォルダを選ぶ」で開いてもらってください。両者が同時編集した場合は、後から保存した方が優先されます（同時編集リスクを完全に避けたい場合は R2 運用がおすすめ）。

Q. 動画のサムネイルはどこのフレーム？

1 フレーム目（動画の頭）です。差し替えたい場合は動画を再アップロードしてください（既存動画のサムネ再生成は自動では走りません）。

Q. データはどこに保存されている？

  • フォルダ運用：あなたが選択したフォルダの中の laycat.project.json と media/ フォルダ
  • R2 運用：Cloudflare R2 のプロジェクトバケット
  • 通知履歴・タブ状態など個人設定：ブラウザの localStorage（別ブラウザ・別 PC には同期されません）

Q. パスワードでプロジェクト自体を保護したい

プロジェクト設定でパスワードを設定すると、laycat.project.json の中身が AES-GCM で暗号化されます。運営（Cloudflare 管理者）でも中身を復号できません。ただし動画本体は暗号化対象外です。
