# LayCAT（旧称：Layna）

単一 HTML で動く、映像制作向けのレビュー／進行管理ツール（ShotGrid ライク）。
動画・画像をアップロードしてディレクターがチェック（OK / リテイク / NG）し、ショット・工程ごとに進行を管理します。
バニラ JS・ビルド工程なしの `laycat.html` 1ファイルで完結します。

## 🔗 公開URL

| 用途 | URL |
| ---- | ---- |
| **本体（LayCAT）** | https://ogshaw03.github.io/lay_cat/ |
| **アクセス管理コンソール** | https://ogshaw03.github.io/lay_cat/access-console.html |

- ルートの `index.html` が本体 `laycat.html` へリダイレクトします（`?クエリ`・`#ハッシュ`はそのまま引き継ぐため、`#/r/…` 等の共有URLもそのまま使えます）。
- Chrome / Edge 推奨（フォルダ共有・接続に File System Access API を使用）。
- アクセス管理コンソールは、ライブサイトにログインできるユーザー（許可メール／ドメイン）を管理する画面です（後述）。

## 主な機能

- 🎬 **レビュー**：動画・画像をアップロードし、フレーム単位でコメント／アノテーション。OK / リテイク / NG / 保留の判定。
- 🗂️ **タブ表示**：タイムライン（更新履歴）／進捗／ショット（ホバー再生）／サブミット／チェック待ち。
- 🧩 **工程（タスク）管理**：ショットごとの各工程を「未着手 / 作業中 / 確認待ち / リテイク / OK」で管理。作業担当・チェック担当を工程単位で設定。ステータス切替はアノテーションウィンドウ／REEL でもプルダウンで行えます。
- 🏷️ **役割の可視化**：各項目が「カテゴリ / ショット / 工程 / タスク」のどれとして扱われているかを、ツリーのアイコン（フォルダ＝カテゴリ／カメラ＝ショット／フィルム＝工程／チェック＝タスク）とページ見出しのチップで明示。進捗タブでは「この階層がショット、その下が工程」という判定を具体例つきで表示（工程かどうかは階層構造から判定）。
- ✍️ **アノテーション**：ブラシ（スムージング）・レイヤー・トレーシングペーパー・ゴースト表示・その場編集。記入→送信でタスクページに反映。
- 🧭 **顔の向きガイド**：正中線を3Dの球体ワイヤーで表示し、ドラッグで顔の角度（ヨー／ピッチ／ロール）を指示。
- ⇄ **比較再生（A/B）**：別の版・カットとアノテーションウィンドウ内で横並び同期再生。
- 🖐️ **クリスタ準拠の画面操作**：Space＋ドラッグ＝パン／Ctrl＋Space（または右ドラッグ）＝ズーム／Shift＋Space＝回転／**V＝左右反転（鏡像）**。
- 💬 **メンション・通知**：`@名前` / `@ロール` でメンション。ベルに通知。
- 📤 **サブミット**：複数カットをまとめて提出。提出ページに固有URLを添付可能。
- 🔒 **アクセス制御**：ログイン許可リスト（下記）＋プロジェクト個別パスワード保護。
- 🎞️ **REEL**：動画クリップを並べて連続再生・提出。

バージョンはヘッダ左上に表示（`laycat.html` 内の `APP_VERSION` で管理。更新のたびに番号を上げます）。

## 使い方（単一HTML版・推奨）

`laycat.html`（＝公開URL）をブラウザで開くだけ。localhost もインストールも不要です。

### データはプロジェクト単位で保存（チーム共有向け）
- 各プロジェクトのデータ（ステータス・コメント・動画）は、そのプロジェクトの**フォルダ内 `laycat.project.json` と `media/`** に保存されます（旧 `layna.project.json` は自動で読み込み・保存時に新名へ移行）。ファイルサーバー上のフォルダを指定すれば、それがチームの「1つの正」になります。
- 新規プロジェクト作成時に**保存先フォルダを指定**します。未設定だと警告が出て、メタ情報は一時的にブラウザ内（localStorage）に保存され、**動画・画像はフォルダ設定まで保存できません**。
- 他の人が作成した（既存の）プロジェクトは、ホームの **「⇄ プロジェクトを接続」** でそのフォルダを開くと自分の一覧に追加されます（ユーザーごとに「使うプロジェクトだけ」を接続）。
- 保存したリールは `reels/reels.json`、個人設定・接続一覧・通知は各自のブラウザ内（localStorage）に保存され、共有データと分離されています。
- ブラウザ内DB（IndexedDB）は**「フォルダへのアクセス権（鍵）」の保持のみ**に使用します。
- フォルダ共有・接続には File System Access API 対応ブラウザ（**Chrome / Edge**）が必要です。

### 画面ごとに固有のURL（作業者間で共有可能）
各タブ／セクション・レビューページ／動画レビュー画面は**固有のURL（ハッシュ）**を持ち、共有すると相手が同じ画面を開けます。ルートの `index.html` はハッシュを保持して `laycat.html` にリダイレクトするため、下記のいずれも公開URLのままそのまま共有できます。

- `#/n/<プロジェクトID>/<タブ>` … タブ（timeline / progress / shots / submit / check / browse）
- `#/n/<ノードID>/…` … タスクページ（工程）
- `#/r/<ノードID>/<バージョンID>` … 動画レビュー（アノテーション）画面
- `#/submit/<サブミットID>` … 提出ページ（提出連絡に添付できる短縮リンク）

## 対応する動画コーデック（.mov / .mp4）

`.mov` ファイル自体は受け入れます（MIME が空でも拡張子で動画と判定）。ただし**サムネイル生成・再生ができるかは中の「コーデック」次第**です。ブラウザ（`<video>` タグ）が再生できるのは H.264 / H.265 / VP9 / AV1 などに限られ、これは LayCAT に限らずどの Web ツールでも共通の制約です。

### Maya の QuickTime（qt）書き出しコーデック別の対応

| Maya QuickTime の書き出しコーデック | ブラウザでの再生 | LayCAT での表示 |
| ---- | ---- | ---- |
| **H.264** | Chrome / Edge / Safari で可 | ✅ サムネ・再生OK |
| Animation（RLE） | 全ブラウザ不可 | ❌ 保存はできるが黒画・再生不可 |
| Photo-JPEG | 全ブラウザ不可 | ❌ 同上 |
| PNG / None(raw) | 全ブラウザ不可 | ❌ 同上 |
| Apple ProRes | 全ブラウザ不可 | ❌ 同上 |

> Maya の QuickTime 書き出しは Animation・Photo-JPEG・ProRes などが選ばれていることが多く、その場合はブラウザで再生できません。

### 確実に動かすには（H.264 で書き出す／変換する）

- **Maya 側**：Playblast／ムービー書き出しのエンコーダを **H.264** に設定する。
- **変換**：`ffmpeg -i input.mov -c:v libx264 -pix_fmt yuv420p -movflags +faststart output.mp4`
  （`-pix_fmt yuv420p` が重要。422/444 だとブラウザで再生できないことがあります）

> 再生できないコーデックの動画は、LayCAT の再生窓の中に「この動画はブラウザで再生できないコーデックの可能性があります（Animation / ProRes 等）。H.264 の mp4 / mov へ変換してください」と常時表示されます。

## アクセス管理

ライブサイト（`https://ogshaw03.github.io/lay_cat/`）では **Google ログイン**で入室可否を判定します（`github.io` 以外＝ローカル/exe/プレビューでは自動的に認証オフ）。

- 許可リストは **`access.json`** で定義します（ライブサイト側に配置）。
  - `authRequired`：認証の有効/無効
  - `adminEmails` / `allowedEmails` / `allowedDomains`：管理者・許可メール・許可ドメイン
- 付与するには `allowedEmails`（またはドメイン）を編集して反映します。**アクセス管理コンソール**（`access-console.html`）から編集することもできます。
- 🔗 **招待リンク（承認不要URL）**：アクセス管理コンソールから発行できます。このリンクで LayCAT を開いた人は、承認を待たずに自動的にメンバーになります（何人でも可・無効化するまで有効）。発行・無効化・削除、参加者の確認／解除がコンソールで行えます。招待データは Firestore（`laynaAccessInvites` ＝リンク、`laynaAccess/invited` ＝参加者）に保存します。リンクを知っている人は誰でも入れるため、配布先は信頼できる範囲に限定してください。
- 各プロジェクトには**個別パスワード**も設定できます。パスワードから導出した鍵（PBKDF2→AES-GCM）で**プロジェクトデータ（コメント・ノート等）を暗号化して保存**します。パスワードが分からないと中身は復号できません（※動画・画像ファイル本体は暗号化対象外。パスワード紛失時は復旧不可）。
