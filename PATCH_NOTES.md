# LayCAT 更新情報

## beta v0.0.5 （2026-07-23）

### 🎞 EXR 連番シーケンス対応（新機能）
- **複数の EXR ファイルを連番として認識してアップロード可能に**（ローカルフォルダ運用のみ）。`＋ 動画` から連番の EXR を選択すると、命名パターン（`base.001.exr` 等）を自動検出して 1 版にまとめます。
- **レビュー UI が mp4 と統一**：連番専用の操作バーは廃止し、動画で使っているタイムライン（フレーム目盛り／IN・OUT／プレイヘッド）・−1F／＋1F／再生／ループ／範囲設定を、そのまま同じ操作で使えます。fps セレクタで再生速度も切替可能。
- **多層 EXR のレイヤー切替＋露出／ガンマ／Depth 調整** を再生中もフレーム毎に反映。レイヤーを切り替えたあと再生しても、そのレイヤーで再生し続けます（従来は再生後に beauty に戻る不具合ありのため修正）。
- **キャッシュ戦略を Adobe After Effects の RAM プレビュー方式に**：アップロード時の全 F 事前生成をやめ、再生時に描画したフレームを LRU 60 F でメモリキャッシュ、成功時に 1F 先をプリフェッチ。スクラブ中はプリフェッチをスキップして UI 応答性を維持します。
- **キャッシュ済みフレームをタイムラインに青ラインで可視化**（REEL 準拠）。レイヤー／露出／ガンマを変えるとキャッシュが自動的に無効化され、青ラインも即座に消えます。
- **「🎞 全キャッシュ」ボタン**（アノテウィンドウ上部）：現在のレイヤー・露出・ガンマ設定で全フレームをまとめてメモリキャッシュに投入。処理中は「キャッシュ中 N/M（クリックで中断）」表示、途中中断可能。完了後は再生・スクラブが完全に途切れなく動きます（AE の "Fully Cached" 相当）。
- **ショートカットキー（Space / ← → / , . / X / V）を連番でも有効化**。動画と同じキーで操作できます。
- タスクページのタイルに紫の `SEQ Nコマ` バッジを表示。削除時は各フレームの実 EXR とキャッシュ JPEG の両方が自動で除去されます。

### 🤝 同時編集リスク低減（保存構造の刷新）
- **プロジェクトデータを「サブミット単位」「ショット単位」でファイル分割保存**するように変更しました。従来は 1 つの `laycat.project.json` に全データを詰めていたため、複数人が同時にサブミットを提出したり別々のショットを編集すると保存内容が上書きで消えるリスクがありました。今回の刷新で、**別のサブミット・別のショットの並行編集はゼロ衝突**になります。
  - サブミットは `submits/<subId>.json` に個別保存、`project.json` 側は `submitIds` のインデックスのみ持ちます。
  - ショットは `shots/<shotId>.json` に「ショット本体＋配下の全タスク（版・アノテ・コメント）」のフルデータを格納。`project.json` は骨格のみ（版・コメントを抜いた軽量版 + `shotIds` インデックス）に。
- **同じショットを 2 人が同時に編集した場合も、双方の追記が保持される 3-way マージ**を実装しました（楽観ロック `_rev`）。保存直前にリモートを再読して他者の更新に気づいたら、版・アノテ・コメント・工程追加を union してから書き込み、DB にも反映します。マージが発生した場合は「ほかの人の更新をショット/サブミット単位で取り込みました」トーストで通知。
- **既存プロジェクトは初回起動時に自動移行**：v<4 のプロジェクトが検出されると起動時に確認モーダルが 1 回だけ表示され、「移行を実行」を押すとその場で新形式に書き出されます（既存データは一切変更されず、shots/ と submits/ が新規に追加されるだけ）。暗号化プロジェクトも同様に対応。
- **保存前のリモート再読を骨格のみに軽量化**：`project.json` 骨格だけを読んで他者の追加ノードを取り込み、shots/submits の並列ロードは省略。10 プロジェクト × 30 ショット環境で毎回の I/O が **300 ファイル並列読込 → 10 ファイル** に削減されます。
- **暗号化プロジェクトの解錠時にも shots/submits を個別復号ハイドレート**するよう修正（従来は骨格しか展開されず、解錠後に版・コメントが空に見える不具合を修正）。暗号化かつ v<4 の旧プロジェクトも、解錠時に移行モーダルが表示されるようになりました。

## beta v0.0.4 （2026-07-22）

### 🗂 タスクタブがブラウザリロードで消える不具合を修正
- 作業中に開いていたタスクタブが、リロード後もそのまま復元されるようになりました。
- 内部的にはハッシュ復元経路でも `openTasks` / `expanded` を localStorage から復元し、URL の対象がタスクなら自動でタブに追加。加えて、暗号化解錠前や R2 の非同期取り込み中で一時的に未解決になった ID をタブ一覧から破壊的に削除しないよう堅牢化しました。
- 「タブを閉じる」「すべて閉じる」ボタンも、現在表示しているプロジェクトのタブだけを対象にするよう修正（別プロジェクトで温存しているタブに意図せず飛ぶ事故を防止）。

### 🎬 mp4 アップロード直後にコーデック警告オーバーレイが誤表示される不具合を修正
- 動画アップロード直後、正しく再生できるはずの mp4 なのに「ブラウザで再生できないコーデックの可能性があります」の警告オーバーレイが出てしまう問題を根本対処しました。
- 内部的には `uploadVersion` 内でアップロード元 File と生成したサムネ Blob から blob URL を作って `storage.urlCache` に事前投入。以降 `renderBody` および `openReview`（FB オーバーレイ）は同期でこの URL を `<video>.src` に設定するため、R2 からの再ダウンロードや src 差し替えの隙間が発生せず、URL が「アップロード完了時点で確定→変わらない」状態になります。
- 加えて、`loadedmetadata` / `canplay` が到達したら誤爆と判定して既存 badge を除去、error 発火から 1.2s 遅延判定と組み合わせて、実際にコーデック非対応（ProRes / Animation 等）の場合のみ警告が出るようになりました。

## beta v0.0.3 （2026-07-21）

### 🎨 EXR フォーマット対応（新機能）
- **多層 EXR をブラウザで直接プレビュー可能に**（Nuke/After Effects なしでレンダー中間ファイルを確認できます）
- 自作 EXR パーサ搭載：ZIP / ZIPS / 無圧縮、HALF / FLOAT、Scanline / Tiled（Nuke・Arnold 書き出し）に対応
- **レイヤー自動判定して可視化**：
  - **RGB カラー**（beauty・diffuse 等）
  - **Depth**（Z）— グレースケール、黒点／白点／反転を手動指定可能
  - **Normal**（X/Y/Z）— カラーマップ
  - **Motion Vector**（角度→色相・強度→彩度）
  - **UV パス**（R=U, G=V）
  - **Position パス**（bbox 正規化）
  - **Cryptomatte**（プレビュー & ランク・ID ハッシュで色分け）
  - 輝度 (Y) / アルファ (A) の単色表示
- **アノテウィンドウにレイヤー切替＋露出／ガンマ／Depth 黒白点／反転** をコンパクトなスライダで搭載（数値直接入力も可能）
- 「🎨 全レイヤー」モーダル：EXR 内の全レイヤーをタイル一覧で俯瞰、クリックで直接開く
- 「🖼 サムネ更新」ボタン：任意レイヤー・設定でタスクページ一覧のサムネイルを更新
- 2 回目以降は **キャッシュ** で即座に開ける
- 非対応：Multi-part / Deep / PIZ・PXR24・DWA 圧縮

### 🌩 Cloudflare R2 保存対応（新機能）
- プロジェクトデータを **クラウドに保存できる** ように（従来のローカルフォルダ運用と併用可能）
- 新規プロジェクト作成モーダルに「ローカルフォルダ / Cloudflare R2」の選択肢を追加
- ホームの「⇄ プロジェクトを接続」に統合、参加可能な R2 プロジェクトを **タイル一覧** で表示
- **メンバー方式アクセス制御**：owner + members に登録された人だけがアクセス可能（`_access.json` 経由・Worker で 403 判定）
- Worker のセキュリティ強化：ファイルサイズ上限（JSON 10MB / メディア 500MB / 絶対 2GB）＋監査ログ
- セットアップ手順は `docs/R2_SETUP.md` 参照（Cloudflare 側の設定必要）

### 📝 タスクページ NOTE パネル（新機能）
- タスクページ右カラムに **リッチテキストのメモパネル** を追加
- 太字／斜体／下線／見出し（H₁・H₂）／文字サイズ／境界ライン
- 手動保存（**Ctrl+S** または「保存」ボタン）— 同期のズレを防ぐため
- 未保存のまま閉じようとすると警告
- 右側スライドではスペース確保のため自動的に非表示

### ✎ アノテーション操作の改善
- 描画／消しゴムを **1 つのトグルボタン**に統合（OFF → 描画 → 消しゴム → OFF と循環）
- ブラシサイズ／消しゴムサイズも **1 つのセレクト**に統合（現在のツールに応じて自動切替）
- **画像・EXR** で送信ボタンを押したとき、その瞬間の見た目（レイヤー切替中の状態＋描画）を **スクショとしてノートに焼き込み**（フレームの概念がない画像でもコメントにスクショが残るように）
- 画像アイテムのフレームコメント欄が正しく表示されるように

## beta v0.0.2 （2026-07-17）

### 再生範囲（IN/OUT）機能を追加
- **アノテーションウィンドウ**と **REEL** の両方に、動画の再生範囲を指定する機能を追加しました。
- タイムスライダで位置を動かして **`＜`** で開始（IN）、**`＞`** で終了（OUT）を設定、**`⌫`** で解除。
- 設定範囲はタイムラインに青の帯でハイライト。
- 再生ボタン押下時、プレイヘッドが範囲外なら自動で IN へジャンプ。
- 範囲設定中に OUT に到達したら、ループONなら IN へ戻ってループ、ループOFFなら OUT で自動停止。
- REEL では複数クリップを跨いだ範囲指定も可能。

## beta v0.0.1 （2026-07-17）

LayCAT の最初のパッチノート付きリリース。これまでに搭載された主な機能をまとめました。

### レビュー・アノテーション
- 動画・画像のフレーム単位のコメント／アノテーション
- ブラシ／消しゴム／顔の向きガイド／トレーシングペーパー／ゴースト表示
- ズーム／パン／回転／**左右反転**（`V` キーでトグル）
- **筆圧検知**（環境設定でON/OFF、既定OFF）
- **`Ctrl + 右クリック` パイメニュー**（描画・消しゴム切替）
- 比較再生（A/B・同期再生）
- 消しゴム操作を含む **Undo/Redo**

### タスク管理
- ショット・工程ごとのステータス管理（プロジェクトごとにカスタム定義可）
- 動画追加時に自動でステータスを「チェック待ち」に設定
- サブミット（複数カット一括提出、全体コメント＋カット個別コメント）
- メッセージ機能（誰でも削除可能、tombstone で復活防止）
- タスクページ／アノテウィンドウ／REEL 各所での動画・メッセージ削除

### 共有・コラボレーション
- フォルダ経由のプロジェクト共有（`laycat.project.json`）
- 3-way マージによる同時編集対応
- `laycat.project.json` 破損時の自動修復（末尾ゴミ除去）
- @メンション・通知

### REEL（シーケンスプレイヤー）
- 複数動画をクリップとして並べて連続再生
- REEL 内でもアノテーション・ステータス切替・左右反転可能

### アクセス制御
- Google ログイン（`access.json` の許可リストに基づく）
- プロジェクト個別パスワード保護（PBKDF2 → AES-GCM）
